Batch / CSV Upload

Upload metadata for many languages at once via CSV + folder structure for images.

CSV Format — Text Metadata

Create metadata.csv saved as UTF-8. First row must be this exact header:

language_code,title,short_description,full_description,video_url,spatial_xr_video_url,nonspatial_xr_video_url

⚠️ Critical quoting rule:

Any field containing a comma or newline MUST be wrapped in double-quotes. This includes short_description and full_description if they contain commas.

✅ Correct example:

en-US,Sleep Timer,"Turn off music, videos, and apps when you sleep.","Full description here, it can be long.",,,
hi-IN,स्लीप टाइमर,"संगीत बंद करें, आसानी से","पूरा विवरण यहाँ",,,

❌ Wrong — short_description has comma but is NOT quoted:

en-US,Sleep Timer,Turn off music, videos when you sleep,Full description,,,

Additional rules:

— video_url must start with https:// or be left blank. Invalid values are auto-skipped to prevent API errors.

— spatial_xr_video_url and nonspatial_xr_video_url are logged but not sent (not supported in text listing API).

— Trailing empty columns (extra commas at end of row) are fine.

— Always run Dry Run first to verify parsed fields before uploading.

Folder Structure — Images & Media

Select the root metadata_images/ folder:

metadata_images/
  en-US/
    icon.png                   ← 512×512 App Icon
    banner.png                 ← 1024×500 Feature Graphic
    screenshots/
      phone/       01.png 02.png …   ← Phone Screenshots (up to 8)
      tablet7/     01.png …          ← 7" Tablet
      tablet10/    01.png …          ← 10" Tablet
      chromebook/  01.png …          ← Chromebook
      xr/          01.png …          ← Android XR
  hi-IN/
    icon.png
    banner.png
    screenshots/
      phone/  01.png
  fr-FR/
    icon.png

— icon.png/jpg and banner.png/jpg matched by name (case-insensitive).

— Screenshots uploaded alphabetically. Omit any subfolder to skip that type.

— Each language gets its own separate edit+commit cycle.

Workflow: Text CSV + Image Folder

Run CSV Text tab for all text fields, then Bulk Images tab for images. Together they cover a full store listing update for all languages at once.

⚡ Connection Test
Before uploading, test that the image upload API is reachable from your browser.

Select the root metadata_images/ folder. The tool reads the folder tree and uploads images per language.

⚠️ Apps Script proxy required for image uploads. Google blocks binary image uploads from browsers (CORS restriction) — text/CSV updates work fine without it. Deploy the proxy once (~3 min) and paste the URL below. Setup guide ▸
One-time Apps Script proxy setup:
1️⃣ Open script.google.com → New project
2️⃣ Delete the default code. Paste the contents of PlayConsoleImageProxy.gs (download below)
3️⃣ Click Deploy → New deployment
   • Type: Web App
   • Execute as: Me
   • Who has access: Anyone
4️⃣ Click Deploy → authorise when prompted
5️⃣ Copy the Web App URL (starts with https://script.google.com/macros/s/...)
6️⃣ Paste it in the field below and click Save
